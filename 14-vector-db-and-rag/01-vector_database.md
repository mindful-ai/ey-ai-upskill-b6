# Vector Databases -- Workshop Handout

## 1. What is a Vector Database?

A **Vector Database** is a specialized database designed to **store,
index, and search high‑dimensional vectors (embeddings)** efficiently.

It is widely used in:

-   Retrieval Augmented Generation (RAG)
-   Semantic search
-   Recommendation systems
-   Image similarity search
-   Fraud detection

Major AI companies such as OpenAI, Google, and Meta rely on vector
search internally.

------------------------------------------------------------------------

# 2. What is a Vector (Embedding)?

A **vector** is a list of numbers representing the semantic meaning of
data.

Example:

Text: "Dogs are loyal animals"

Embedding: \[0.21, -0.84, 0.67, 0.19, ...\]

These embeddings are generated by **embedding models** such as:

-   OpenAI Embeddings
-   Sentence Transformers
-   BERT

Similar sentences produce similar vectors.

Example:

  Sentence               Similarity
  ---------------------- --------------
  Dogs are loyal         Very similar
  Puppies are friendly   Similar
  Cars need fuel         Different

------------------------------------------------------------------------

# 3. Why Traditional Databases Fail

Traditional databases like MySQL or PostgreSQL rely on **exact
matching**.

Example:

SQL Query:

SELECT \* FROM documents WHERE title = "AI research"

But AI systems need **semantic matching**.

Example query:

"How do neural networks learn?"

Expected results:

-   Deep learning training
-   Backpropagation explained

Vector databases enable this capability.

------------------------------------------------------------------------

# 4. How Vector Databases Work

Vector search typically involves **four steps**.

## Step 1 -- Convert Data to Vectors

Text or images are converted to embeddings.

Example Python:

``` python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")

text = "Python is used for AI"
vector = model.encode(text)
```

------------------------------------------------------------------------

## Step 2 -- Store Vectors

Vector DB stores:

-   vector
-   original document
-   metadata

Example:

  id   vector          text            metadata
  ---- --------------- --------------- ----------
  1    \[0.12,0.34\]   Python for AI   blog
  2    \[0.88,0.11\]   ML basics       tutorial

------------------------------------------------------------------------

## Step 3 -- Indexing

Vectors often contain **hundreds or thousands of dimensions**.

To search efficiently, vector databases use:

-   HNSW (Hierarchical Navigable Small World)
-   IVF (Inverted File Index)
-   Product Quantization
-   ScaNN

These allow **Approximate Nearest Neighbor (ANN) search**.

------------------------------------------------------------------------

## Step 4 -- Similarity Search

Process:

1.  Convert user query to vector
2.  Search nearest vectors
3.  Return most similar documents

Example:

Query: "What is deep learning?"

Results:

Doc 12 -- similarity 0.92\
Doc 33 -- similarity 0.89\
Doc 5 -- similarity 0.87

------------------------------------------------------------------------

# 5. Similarity Metrics

Common distance metrics:

### Cosine Similarity

Measures angle between vectors.

Range: - 1 = identical - 0 = unrelated

### Euclidean Distance

Straight‑line distance.

Lower distance = more similar.

### Dot Product

Used by some embedding models.

------------------------------------------------------------------------

# 6. Architecture of a Vector Search System

Typical pipeline:

User Query ↓ Embedding Model ↓ Query Vector ↓ ANN Index Search ↓ Top‑K
Similar Documents ↓ Returned Context

# 7. Vector Databases in RAG Systems

Vector databases are the **core component of Retrieval‑Augmented
Generation (RAG)**.

Pipeline:

Documents ↓ Chunking ↓ Embeddings ↓ Vector DB ↓ Similarity Retrieval ↓
LLM ↓ Answer

Without vector retrieval, LLMs often **hallucinate**.

# 11. Example: Simple End‑to‑End Vector Search

``` python
from sentence_transformers import SentenceTransformer
import faiss
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

documents = [
"Python is used in AI",
"Deep learning uses neural networks",
"Cars run on fuel"
]

doc_vectors = model.encode(documents)

dimension = doc_vectors.shape[1]
index = faiss.IndexFlatL2(dimension)

index.add(np.array(doc_vectors))

query = "How do neural networks work?"
query_vector = model.encode([query])

distances, indices = index.search(np.array(query_vector), k=2)

print(indices)
```
